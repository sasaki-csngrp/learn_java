C言語版の`Hello.c`を作成しました。C言語でもポリモーフィズム相当を実装できます。

この実装では、以下の方法でポリモーフィズムを実現しています：

1. **インターフェース相当の基底構造体**: `Human`構造体を定義し、関数ポインタでインターフェースを表現
   - `sayHello`と`destroy`を関数ポインタとして定義

2. **構造体の埋め込み**: `Japanese`と`American`構造体の最初のメンバーとして`Human`構造体を埋め込み
   - これにより、`Japanese*`や`American*`を`Human*`に安全にキャスト可能

3. **ポリモーフィズムの実現**: 同じ`Human*`型のポインタに、異なる実装（`Japanese`/`American`）を代入可能
   - 同じインターフェース経由で呼び出すが、実際に実行されるのは各実装の関数

4. **実行時の多態性**: 関数ポインタにより、実行時に正しい実装が呼ばれる

コンパイルと実行の例：
```bash
gcc -o Hello Hello.c
./Hello
```

実行すると：
```
こんにちは。私の名前は太郎です。
Hello. My name is John.
```

**C言語でのポリモーフィズムの特徴**：
- 関数ポインタにより、実行時の多態性を実現
- 構造体の埋め込みにより、型安全性をある程度保証
- Javaのような言語サポートはないため、手動で実装が必要
- メモリ管理も手動で行う必要がある

C言語でもポリモーフィズムは実現可能です。この実装は、Javaのインターフェースとポリモーフィズムの概念をC言語で表現したものです。
